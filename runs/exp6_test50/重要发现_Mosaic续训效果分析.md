# 🔬 重要实验发现：Mosaic 重开续训效果分析

> **重要性**: ⭐⭐⭐⭐⭐ | **实验**: exp6_test50 | **日期**: 2026-06-29

---

## 一句话结论

**原始 1000 epoch 训练在 ep16 关闭 Mosaic 后，984 轮无增强训练导致模型严重过拟合到单图分布。重新打开 Mosaic 仅 50 epoch（lr=0.0003），val mAP50-95 从 0.374 飙升至 0.417（+0.043），test 从 0.371 升至 0.415（+0.044）。所有类别无一下降。**

---

## 实验设置

| 项目 | 原始训练 (enhanced_fold0_1000ep-10) | Exp6 |
|------|--------------------------------------|------|
| 模型 | yolo11n-obb-p2 | 加载原始 best.pt |
| close_mosaic | **15** (ep16 后永久关闭) | **30** (ep1-30 开启) |
| lr | auto (MuSGD, lr0=0.01) | MuSGD, lr0=0.0003 |
| epochs | 1000 | 50 (续训) |
| Mosaic 开启 epoch 数 | 15 / 1000 (1.5%) | 30 / 50 (60%) |

## 关键发现

### 1. 增益与样本量成反比

```
样本量        AP50-95 增益
──────────────────────────
<500 (Tail)   +0.033 ~ +0.105  ← 最大受益
500-5000      +0.040 ~ +0.058
20000+ (Head) +0.019 ~ +0.026  ← 受益最小
```

**规律**: 样本越少、场景越单一的类别，Mosaic 收益越大。Head 类（Small Car/Van）本身样本多场景丰富，mosaic 用处不大；Tail 类（Bus/Excavator/Tractor）样本少且总出现在固定场景，mosaic 把它们拼到随机背景中 → 逼模型学习"目标本身"而非"目标所在场景"。

### 2. Val 和 Test 高度一致

Val +0.043, Test +0.044。增益是真实的，不是验证集过拟合。

### 3. close_mosaic=15 是原训练最大失误

原训练 1000 epoch 中只有前 15 epoch（1.5%）有 Mosaic。剩余 984 epoch 模型在无增强的"死数据"上训练，学到了单图统计偏差。模型并非"收敛"，而是"学僵了"——证据是重新打开 Mosaic 后仅 50 epoch（lr 仅 0.0003）就暴涨 0.043。

### 4. Bus 是最典型案例

Bus 只有 171 个训练样本，且总出现在道路、车站等固定场景。原始训练中模型学到的是"那个场景 + 那个形状 = Bus"。Mosaic 把 Bus 拼接到停车场、十字路口等随机背景中，逼模型解耦"场景"和"目标"。效果：Val +0.097, Test +0.105。

### 5. 增益上限预估

50 epoch 仍在上升趋势中（val loss 持续下降）。如果续训 500-1000 epoch，保守估计 mAP50-95 可达 0.43-0.45。

---

## 经验教训

1. **close_mosaic 不宜过早**: YOLO 社区常见 close_mosaic=10~15，但对于遥感小目标+长尾分布，30+ 更合适。
2. **Mosaic 对长尾类别有独特价值**: 不仅是数据增强，更是一种"场景解耦"手段。
3. **Loss 不反弹 ≠ 已收敛**: 原训练 val loss 降到 0.816 且不再反弹，但它是在无增强的偏差数据上"收敛"的。
4. **续训比调参更有效**: Exp2 换了高 lr 费了 360 epoch 才到 0.388，而 Exp6 仅用原 lr+50 epoch 就到 0.417。
