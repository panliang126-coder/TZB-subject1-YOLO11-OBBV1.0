## 二、核心发现：cls_loss = 0（严重 BUG）

### 现象

| 实验 | Focal Loss | cls_loss（epoch 1） |
|------|-----------|---------------------|
| debug_clean | ❌ 关闭 | **4.63** ✅ 正常 |
| 当前实验 (-5) | ✅ γ=2.0, α=0.25 | **1e-05 → 0** ❌ 异常 |

**Focal Loss 把分类 loss 直接压到零，差了 46 万倍。**

### 根因分析

```
focal_gamma=2.0, focal_alpha=0.25
         │
         ▼
   cls_loss ≈ 0  （分类梯度消失）
         │
         ├── 分类头无法学习 → P 剧烈波动（0.01↔0.47）
         │
         └── mAP 增长极慢（12 epoch 才 0.076）
```

OBB 遥感场景下，小目标占比高、类别不平衡严重，`γ=2.0` 的标准 Focal Loss 参数过于激进，将大量易分样本的 loss 压制到浮点精度以下。

### 3.1 ✅ 健康的指标

| 指标 | 趋势 | 变化 | 评价 |

| val < train loss | val_box=1.005 < train_box=1.189 | — | 未过拟合，增强有效 |

### 3.2 ⚠️ 异常指标

| 指标 | 现象 | 原因 |
|------|------|------|
| cls_loss | 始终为 0 | Focal Loss γ 过大 |
| Precision | 0.01 → 0.47 → 0.08 → 0.19 → 0.10 | cls_loss=0 导致分类头不可靠 |
| mAP50 | 0.003 → 0.076（12 epoch） | 增速过慢，正常应在 20-30 epoch 到 0.3+ |

### 3.3 Epoch 4-5 假阳性事件

P 飙升到 0.47，但 R 只有 0.11。说明模型只检出极少数高置信度框，大多数目标被漏检。这是分类信号缺失导致模型「过度保守」的典型症状。
